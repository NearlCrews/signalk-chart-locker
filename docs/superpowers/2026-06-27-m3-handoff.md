## Repo map

- `src/`, `test/`: the Node plugin. Lifecycle, the `signalk-container` consumer, and the
  in-process bridge `globalThis.__signalk_binnacle_routeOnWater`.
- The `container/` Rust crates are one Cargo workspace (`container/Cargo.toml`): engine, gpkg,
  localprovider, router, and storage-spike, sharing a lock, a target dir, and the root
  `container/.cargo/config.toml` (the FMA determinism flag).
- `container/engine/`: the Rust engine (the channel-router port) and the parity corpus. Pure
  geometry, no I/O dependencies. The orchestrator is split into `channel_router.rs`,
  `water_index.rs`, and `snap.rs`, with the navigable-grid build in a `GridBuild` stage struct.
- `container/gpkg/`: the shared pure-Rust GeoPackage and WKB decoder (`binnacle-gpkg`), used by
  localprovider and storage-spike.
- `container/localprovider/`: reads a region GeoPackage with `rusqlite` plus the `binnacle-gpkg`
  decoder and implements the engine `Provider` trait. The `testutil` feature exposes a
  `StoreBuilder` for tests and is excluded from release builds.
- `container/router/`: the axum service (`/health`, `/regions`, `POST /route-on-water`). It
  selects `LocalProvider` when `BINNACLE_REGION_STORE` names a store, `UnavailableProvider`
  (declines `fetch-failed`) on an open error, and `EmptyProvider` (declines `no-coverage`)
  when no store is set.
- `container/storage-spike/`: the Milestone 1.5 proof, kept as a historical record.
- `container/prep/`: the GDAL prep container. `prep_region.py` (build a region store),
  `data_parity.py` (compare a store to the live NOAA ENC), `Dockerfile`, `README.md`.
- `docs/superpowers/specs/`, `plans/`, `reviews/`, `decisions/`, `roadmap/`: the design spec,
  the per-milestone plans, the review records, the resolved decisions, and the cross-plugin
  migration roadmap.

## How to resume: build and test

```
npm test && npm run typecheck && npm run lint && npm run build
cd container && cargo test --workspace && cargo test -p binnacle-localprovider --features testutil
cd container && cargo clippy --workspace --all-targets -- -D warnings \
  && cargo clippy -p binnacle-localprovider --all-targets --features testutil -- -D warnings
cd container && cargo build --release --bin router
```

First Rust builds on a Pi are slow: allow a long timeout.

## How to resume: build a region store

```
podman build -t binnacle-prep container/prep
# download the region's ENC cells (public domain), then unzip:
#   https://www.charts.noaa.gov/ENCs/<CELL>.zip   (each zip has ENC_ROOT/<CELL>/<CELL>.000)
podman run --rm -v /path/to/data:/work binnacle-prep \
  --enc-dir /work/enc --out /work/region.gpkg \
  [--boundaries /work/admin0.geojson --country-field ADM0_A3] [--osm /work/osm_water.gpkg]
```

The usage band comes from the cell name's third character (1 overview through 6 berthing).

## How to resume: run the router on a store

```
BINNACLE_REGION_STORE=/path/region.gpkg ROUTER_PORT=8080 container/router/target/release/router &
curl -s -X POST http://127.0.0.1:8080/route-on-water -H 'content-type: application/json' \
  -d '{"from":{"latitude":37.80,"longitude":-122.42},"to":{"latitude":37.79,"longitude":-122.39},"draftMeters":2.0,"safetyMarginMeters":0.5,"standoffNm":0.02,"borderAware":false}'
```

Request fields: `from`, `to` (each `{latitude, longitude}`), `draftMeters`,
`safetyMarginMeters`, `standoffNm`, `borderAware`, and optional `homeCountryId`, `corridor`,
`bboxAnchors`, `maxSnapMeters`, `deadlineMs`. Response is
`{ok:true, waypoints:[{latitude,longitude}], usedTileWater, borderFallback}` or
`{ok:false, reason}` where reason is one of `no-coverage no-path deadline unsnappable land-leg
fetch-failed` (or `router-unavailable` from the Node bridge on a transport failure).

## Gotchas worth knowing before you continue

- The rolling and Alpine GDAL builds of the S-57 driver segfault (exit 139) on valid NOAA
  cells. Keep the prep image on the pinned stable tag above.
- The runtime image must stay free of GDAL, GEOS, PROJ, and SpatiaLite. GeoPackage reads use
  `rusqlite` with the `bundled` feature plus the pure-Rust WKB decoder only. GDAL lives in the
  prep image alone.
- `LocalProvider` honesty: a successful but empty query returns present-but-empty so the
  engine declines `no-coverage`, and a SQL or decode error returns `None` so the engine
  declines `fetch-failed`. An undecodable geometry must fail loud, never drop an obstacle.
- The dev shell here is zsh (an unquoted variable does not word-split), a foreground `sleep`
  is blocked, and backgrounding the router inside a single shell call can exit 144: verify on
  a fresh port, or run it as a managed background task.
- `osm_water` is the engine's tile-water mask: when present it gates navigability, so the
  source must be a water layer that includes the navigable ocean and bay (the osmdata
  water-polygons split product), not a raw `natural=water` Overpass query, which is inland
  water only. A wrong or partial `osm_water` makes the router decline `no-path` (the safe
  failure), so such a store is not routable even though it ingested cleanly. An empty
  `osm_water` (an ENC-only store) is fine: the engine routes off the ENC depth areas with
  `usedTileWater: false`. To get a green route, pick endpoints actually inside a deep depth
  area (use `ST_PointOnSurface`, not `ST_Centroid`, which can land in a hole on a multipolygon)
  and reasonably close, so the padded route bbox resolves a grid: an `ok: true` route over
  US5CA13M between two interior deep points returns 3 waypoints with `usedTileWater: false`.
- The routing store is polygon-only: the `LocalProvider` WKB reader decodes Polygon and
  MultiPolygon and fails the whole band to `None` on anything else, a fail-loud safety backstop
  that never silently drops an obstacle. S-57 LNDARE carries point and line features too, so
  prep filters the ENC ingest to area geometry (`OGR_GEOMETRY LIKE '%POLYGON'`). Without that
  filter a real cell's MultiPoint or MultiLineString land features make every request decline
  `no-coverage`. Locked by a localprovider regression test.
- Border-aware boundaries source: RESOLVED to Marine Regions EEZ (`country_id = iso_sov1`), not
  admin-0 land. Admin-0 polygons are land and cover no navigable water, so border-aware was a
  silent no-op (`homeCountryId` USA versus CAN gave the identical route over the all-US bay). EEZ
  tiles the ocean by country and covers the water (verified: an offshore and a territorial-sea
  point are in the US EEZ, the inner bay is not), and it decodes through the reader. A hermetic
  test locks the mechanism, and a live test on a real US/Canada border (Haro Strait, NOAA cell
  US3WA1EF, with the Marine Regions US and Canada EEZ) confirms it end to end: the same US-side
  route stays in home water for `homeCountryId: USA` (`borderFallback: false`) and is blocked as
  foreign for `CAN` (`borderFallback: true`). See
  `docs/superpowers/decisions/2026-06-27-border-aware-boundaries-source.md`.
- Prep now clips OSM water and boundaries to the ENC extent plus a 0.1 deg margin
  (`enc_region_bbox` plus `ogr2ogr -clipdst`), so a regional or global source does not write the
  whole world into a region store. A global Natural Earth ocean clipped to one harbour cell
  becomes one `osm_water` row.
